# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

> 이 저장소는 **OURS HUB / OURS TRADING 웹사이트** 프로젝트의 작업 공간입니다.
> 단, 현재 코드베이스에는 기존 Python CLI(`codex-maintainer-kit`)가 남아 있습니다. 웹사이트 코드를 추가하기 전까지는 아래 "현재 코드베이스" 섹션의 명령이 유효하며, 웹사이트 코드가 추가되면 이 문서의 명령 섹션을 갱신해야 합니다.

## 프로젝트 목표 (OURS HUB / OURS TRADING)

- OURS HUB는 **한국발 소싱/수출 플랫폼**입니다. 핵심 버티컬은 K-뷰티와 한국 소비재입니다.
- 주요 타깃은 **러시아/CIS 및 국제 도매 바이어**입니다.
- 웹사이트의 목적은 판매가 아니라 **견적 요청(Quote Request), 공급사/제품 문의 유도, 신뢰 구축**입니다.
- **결제/체크아웃 기능은 없습니다.** 모든 견적은 매니저가 확정합니다. 사이트나 AI가 가격·납기·거래 조건을 확정하는 흐름을 만들지 마세요.
- **Yuna AI**는 "AI임을 명시한(disclosed)" 앰버서더/컨시어지로 활용할 수 있습니다. 단, 가격·MOQ·납기 등 비즈니스 조건은 반드시 매니저 확인을 거친다는 점을 UI 문구와 로직 양쪽에 반영해야 합니다.

## UX / 디자인 원칙

- **사진 중심(photo-rich), 깔끔한 레이아웃**: 올리브영(Olive Young) 스타일의 커머스/카탈로그 감성을 지향합니다.
- CTA(견적 요청, 문의하기)는 **읽기 쉽고 눈에 띄게**, 화면을 답답하게 채우지 않습니다(not cramped). 여백을 충분히 둡니다.
- 모든 주요 흐름의 종착점은 견적 요청 또는 문의 폼입니다. 장바구니/결제 UI 패턴을 흉내 내지 마세요.
- 신뢰 요소(회사 소개, 수출 실적, 인증, 연락처)를 눈에 띄는 위치에 배치합니다.
- **다국어**: 한국어와 러시아어가 중요합니다. 텍스트는 하드코딩하지 말고 처음부터 i18n 구조(언어별 리소스 분리)로 작성하세요. 러시아어 텍스트는 같은 의미라도 한국어/영어보다 길어지는 경우가 많으므로 레이아웃이 깨지지 않는지 확인합니다.

## 코드 컨벤션

- 현재 Python 코드: `ruff` (line-length 100, target py310) 기준을 따릅니다. 타입 힌트와 `from __future__ import annotations` 패턴을 유지합니다.
- 웹사이트 코드 추가 시:
  - 사용자 노출 문자열은 모두 i18n 리소스로 분리합니다 (ko / ru, 필요 시 en).
  - 컴포넌트·페이지는 작게 유지하고, 견적/문의 폼 로직은 재사용 가능한 모듈로 분리합니다.
  - 이미지가 많은 사이트이므로 이미지 최적화(지연 로딩, 적절한 포맷/크기)를 기본으로 합니다.
- 커밋은 작고 집중적으로, 변경 이유가 드러나는 메시지로 작성합니다.

## 안전 / 승인 경계 (반드시 준수)

명시적 승인 없이는 절대 하지 않습니다:

- **실제 아웃리치 금지**: 실 바이어/공급사에게 이메일·메시지 발송 금지. 초안 작성까지만 합니다.
- **공개 게시 금지**: SNS, 게시판, 외부 서비스에 콘텐츠를 게시하지 않습니다.
- **자격 증명 변경 금지**: API 키, 비밀번호, 계정 설정을 수정하지 않습니다.
- **프로덕션 배포 금지**: 명시적 승인 전에는 프로덕션에 배포하지 않습니다.
- Yuna AI 관련: AI가 가격/조건을 "확정"하는 것처럼 보이는 문구·기능을 만들지 않습니다. 항상 "매니저 확인 후 회신" 흐름으로 연결합니다.

## 빌드 / 테스트 명령 (현재 코드베이스 기준)

```bash
# 개발 환경 설치
python -m pip install -e '.[dev]'

# 전체 테스트
pytest -q

# 단일 테스트 파일 / 단일 테스트
pytest tests/test_cli.py -q
pytest tests/test_core.py::테스트함수명 -q

# 린트
ruff check src tests
```

- CLI 진입점: `cmtk` (`src/codex_maintainer_kit/cli.py`의 Typer 앱). 핵심 로직은 `src/codex_maintainer_kit/workflows.py`의 `render_*` 함수들입니다.
- 웹사이트 스택이 추가되면(예: Next.js/Vite 등) 이 섹션에 해당 dev/build/test 명령을 추가하세요.

## 배포 주의사항

- 프로덕션 배포는 **항상 사람의 명시적 승인 후**에만 진행합니다. 스테이징/프리뷰까지는 가능하되, 그 사실을 명확히 보고합니다.
- 배포 전 체크: ko/ru 두 언어 모두 주요 페이지 렌더링 확인, 견적/문의 폼 제출 동작 확인, 모바일 레이아웃 확인.
- 환경 변수·시크릿은 코드에 넣지 않고 배포 환경 설정으로만 관리합니다.

## 흔한 함정 (Common Pitfalls)

- **체크아웃을 만들려는 실수**: 이 사이트에는 결제가 없습니다. "Buy now" 대신 "견적 요청 / Request a Quote"입니다.
- **AI가 조건을 확정하는 문구**: Yuna AI 응답·UI 카피에 가격/납기를 단정하는 표현이 들어가지 않게 주의합니다.
- **러시아어 텍스트 길이로 인한 레이아웃 깨짐**: 버튼·내비게이션은 긴 러시아어 문자열 기준으로도 확인합니다.
- **하드코딩된 한국어 문자열**: 나중에 i18n으로 빼기 어렵습니다. 처음부터 리소스 파일로 분리합니다.
- **이 저장소의 이중 정체성**: 기존 Python CLI 코드와 웹사이트 코드가 혼재할 수 있습니다. 작업 전 어느 쪽을 다루는지 확인하고, CLI 쪽을 건드릴 때는 `pytest -q`로 회귀를 확인합니다.
- **pytest 경로**: `pyproject.toml`의 `pythonpath = ["src"]` 설정에 의존하므로, 테스트는 저장소 루트에서 실행해야 합니다.
